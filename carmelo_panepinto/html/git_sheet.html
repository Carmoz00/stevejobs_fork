Istruzioni di Git

  • git init: Inizializza una repository
  • git status: Per vedere i file nella staging area (file non tracciati, modificati o pronti per il commit)
  • git remote add origin [URL]: Collega la repository alla repository GitHub
  • git clone [URL]: Clona un progetto GitHub sul locale
  • git push origin master (o nome del branch): Carica i file su GitHub
  • clear: Pulisce lo storico del terminale
  • git add [nome_del_file]: Per aggiungere un file alla staging area della repository
  • git add *: Per aggiungere tutti i file della cartella alla staging area della repository
  • git commit -m [nome_etichetta]: Dare un commit (etichetta) ad un file (nuovo o aggiornato)
  • git log: Per vedere la lista dei commit nella repository
  • git reset [hash_commit]: Resetta la repository al commit specificato
  • git checkout [nome_file]: Scarta le modifiche del file e ripristina la versione pi√π recente dal repository
  • git checkout [nome_branch]: Cambia in un branch
  • git branch: Lista i branch
  • git branch [nome]: Crea un nuovo branch
  • git merge: Unisce le modifiche del branch modificato al branch corrente (spesso il master)
